# Telegram Business Bot

Telegram-бот для бизнеса с каталогом товаров, системой заказов (FSM), FAQ, админ-панелью и автоматической очисткой старых заказов.

## Функциональность

### Для пользователей:
- 📋 **Каталог товаров** — просмотр товаров с описанием и ценами
- 🛒 **Оформление заказов** — пошаговый процесс через FSM (Finite State Machine)
- ❓ **FAQ** — ответы на частые вопросы
- 📞 **Контакты** — информация о компании
- 📦 **Мои заказы** — история заказов пользователя

### Для администраторов:
- 🔐 **Админ-панель** — управление через инлайн-кнопки
- ➕ **Добавление товаров** — с фото и описанием
- 📋 **Управление товарами** — просмотр, редактирование, удаление
- 📦 **Управление заказами** — просмотр всех заказов, отметка как выполненные
- 🗑 **Очистка старых заказов** — автоматическая и ручная

## Технологии

- **Python 3.9+**
- **aiogram 3.x** — современный фреймворк для Telegram Bot API
- **aiosqlite** — асинхронная работа с SQLite
- **APScheduler** — планировщик для фоновых задач
- **python-dotenv** — управление конфигурацией через .env

## Установка

1. Клонируйте репозиторий:
```bash
git clone <repository-url>
cd <project-name>
```

2. Установите зависимости:
```bash
pip install -r requirements.txt
```

3. Создайте файл `.env` (скопируйте из `.env.example`):
```bash
cp .env.example .env
```

4. Отредактируйте `.env`:
```
BOT_TOKEN=your_bot_token_here
ADMIN_ID=your_telegram_id_here
DATABASE_PATH=business_bot.db
ORDER_RETENTION_DAYS=30
```

## Запуск

```bash
python -m bot.main
```

## Особенности реализации

### FSM (Finite State Machine)
Процесс заказа реализован через машину состояний:
1. `entering_name` — ввод имени
2. `entering_phone` — ввод телефона (с валидацией)
3. `entering_address` — ввод адреса
4. `confirming_order` — подтверждение заказа

### Валидация
- **Телефон** — проверка формата через регулярные выражения
- **Цена** — проверка положительного числа
- **Длина полей** — минимальная длина для имени, адреса, описания

### Админ-панель
- Доступ через фильтр `AdminFilter` (проверка `user_id`)
- Полностью кнопочный интерфейс (без текстовых команд)
- Подтверждение при удалении

### Автоматическая очистка
- Запускается ежедневно в 3:00
- Удаляет выполненные заказы старше `ORDER_RETENTION_DAYS` дней

## База данных

### Таблица `products`
- `id` — ID товара
- `name` — название
- `description` — описание
- `price` — цена
- `image_url` — URL изображения
- `created_at` — дата создания

### Таблица `orders`
- `id` — ID заказа
- `user_id` — ID пользователя Telegram
- `user_name` — имя клиента
- `phone` — телефон
- `address` — адрес доставки
- `product_info` — информация о товаре
- `total_amount` — сумма заказа
- `status` — статус (pending/completed)
- `created_at` — дата создания
- `completed_at` — дата выполнения
